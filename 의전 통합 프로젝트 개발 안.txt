 

기본적인 Layout 포맷을 가지고 진행해야 하고 추가 사항은 모조리 차후에 

  └ 기획 단에서 쳐낼 수 있는 부분을 맞춰야 함. (심 팀장 님 및 기획 자와 회의)

  └ 기존 메뉴 구성과 통합 시스템 메뉴 구성을 추리고 동일한 메뉴 구성에 있는 작업 필드를 빠르게 구성 (그 후 Member 구성 처리 및 세션 처리 진행)

  └ 기존에 사용하던 쿼리는 그대로 사용 할 수 있도록 진행. (작업 단순 및 속도 개선)

  └ 관리 단 (장례 현황 후 순위 - 일정 제 조정 필요) 구성을 빨리 끝내야 다른 작업 처리가 원할 해짐

  └ Member 레벨 링 구성 파악 (접근 페이지 및 작업 확인 필요)

  └ 관리자 페이지 (견적 구성 때 파악 안되었음.), 장례 설명 페이지(자동화할 것인지 폼 페이지에 데이터 뿌릴 것인지 구성 잡지 않음) 추가에 따라 일정 조정은 필요함.

  └ 기존의 Native App에서 데이터(Api) Result 페이지 확인해 쿼리 추출 (재고 카운트(행사출고, 본부 입고 승인) 및 비품 회수).

   

기본 레이아웃 작업 및 필드 구성을 2명 (초급)에게 일임.

  └ 페이지의 고정 레이아웃 (Top, Bottom, Nav), 각 Contents의 list, write, edit 구성을 처리.

  └ list, edit 페이지 쿼리(난이도 없는 단순 구성일 경우와 기존에 사용되는 쿼리)작업 및 데이터 값 구현.

  └ Publishing 된 페이지에 Data 값 구현.

  └ 공지사항, 서류 양식(자료실 ? ) 작업.

  └ Pagination 이나 경고 창(팝업) 밑작업 필요.
