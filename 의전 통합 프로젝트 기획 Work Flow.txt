1. 통합 사이트 메뉴 구성과 메뉴별 구성 컨텐츠 fix
  └ 구 사이트 대비 요청 사항 파악. 

2. 구 사이트 메뉴와 통합 사이트의 메뉴 Matching
  └ 통합 사이트에 이전 할 것과 버려야 할 것을 명확히 재시 후 Fix

3. 통합 사이트 메뉴 별 Process 흐름도 작성
  └ 구 사이트 구성 Rule (입력 필드, validation 등) 이전 여부 확인.
  └ 신규 메뉴나 구성 컨텐츠 접근권한, 입력 필드, validation 및 다른 메뉴와 연동 여부등
      추가 Rule 작성
  └ Pc와 모바일(Tablet) 분리의 상황에서 특히 클라이언트 (장례 상담)에게 노출 되는 부분과
     재고 관리 부분의 분리를 명확히 분리해서 보안 문제나 재고 부분의 외부 노출이 없도록 정리    

4. 통합 사이트 U.I (Layout) 기획
  └ 재고 관리와 외부 노출 페이지의 차별이 필요.

5. 통합 사이트 디자인 (공통 색상 4~5가지, 버튼 ( ex : 삭제는 경고의 의미 부여로 붉은 색 계열) ) 기획
  └ 디자인 쪽과 같이 진행. 초기엔 안함.

6. 위 상황의 정리 후 Information Architecture 작성
